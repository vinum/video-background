File structure and files to upload

In this chapter I'll give you a brief explanation of the files you have in the download package and which files do you need to upload.

In the download package you have the following:

  • flash folder - This folder contatins the flash component of the script. It has a fla, an swf and an as file inside. You just need to upload the swf file but it has to be in the flash folder.
  • jquery folder - Here you'll find the jquery version 1.6 library and two other helping libraries. You shouln't make any changes to any of these files.
  • js folder - On this folder you have the actual javascript file that will add and control the video background on your site.

You'll need to upload the jquery and js folders and all of it's contents and also the swf file from the flash folder for the script to work properly.

STEP 1 - importing javascript libraries

So the first to do is to import all you'll need for the script to run. What will you need to import:

  • jquery library - used in the backgroudn_video script
  • modernizr - used to check if the client browser supports video and which formats does it support

Having this set up you just have to define the paremters now.

You have a lot of paremeters available, these are the control ones:

  • autoplay - set "true" if you want the video to autoplay as soon as the user enters the site or "false" if not
  • loop - set "true" if you want the video to start again when reaches the end or "false" if not

Having this set and well toughted you should now prepare the video you want to show up on your site's background. The script accepts multiple formats for the video.

 

Why should I include multiple formats for the same video?

The answer is simple, compatibility. If you want your background video to be shown in all systems and browsers you shouldn't have only one format for the video.

Which formats should I have?

I recommend you to have 2 formats, mp4 (the most used and compatible for most of the systems) and ogg or webm. Note that the webm don't work on some linux servers, so if your server has linux you should consider ogg instead of webm.

Flash component can only use 2 of the 4 formats, those are the mp4 and flv, you don't need to have an flv if you have an mp4 format. Html 5 component uses mp4, ogg or webm.

 

So going back to the implementation and assuming that you already have at least two formats for your video you just have to define them in the parameters, you have the following:

  • mp4, flv, webm, ogg - These are used for all personal computers as they have better processing and internet speed you can have slightly better quality than in other systems.
  • mp4Mobile, flvMobile, webmMobile, oggMobile - These are used for all mobile devices as they have lower processing and internet speed you should have lower quality videos.
  • format - It's very important for you to define the aspect ratio your video has. It's default is 16:9 as it's the most used but if you're video has a different one or you're not sure about it open it with a media player on your pc and measure it size, let's suppose it has 1360x600 you can set the format = "1360:600". This will be used to resize the video according to the window's size.
  • formatMobile - Same as format but for the mobile videos (usually the same as format)

 

You need to have to have formats for mobile even if they're the same as the normal. For example, if you have a mp4 and ogg version and feel that you don't need or want to reduce it's quailty for mobile you need to set mp4Mobile the same path as mp4 and same for ogg.

 

Now you just have to define the parameters correctly, on my example I have a mp4 and ogg format with 16:9 aspect ratio, also I want it to autoplay, to loop and not be muted initialy, so I'll have:

 $("body").video_background({  format:"16:9",
                               mp4:"videos/sample.mp4" ,
                               ogg:"videos/sample.ogg", 
                               formatMobile:"16:9",
                               mp4Mobile:"videos/mobile/sample.mp4" ,
                               oggMobile:"videos/mobile/sample.ogg",
                               loop:"true",
                               autoplay:"true",
                               muted:"false"
                             });

Control functions

There a few control functions that you can call by javascript or on a button click to interact with the background video:

  • background_play() - Plays the video
  • background_pause() - Pauses the video
  • background_toggle_play() - Plays/Pauses the video according to it's state
  • background_mute() - Mutes the video
  • background_unmute() - Unmutes the video
  • background_toggle_sound() - Mutes/Unmutes the video according to it's state
  • background_hide() - Hides the video
  • background_show() - Shows the video
  • background_toggle_hide() - Hides/Shows the video according to it's state
  • background_rewind() - Rewinds the video to the start
